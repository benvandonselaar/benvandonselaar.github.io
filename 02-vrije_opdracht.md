# Personal assignment
I chose to learn variant annotation because it is a relevant bioinformatics skill for working with genomic data in biomedical research. Variant annotation adds biological context to genetic variants by linking them to genes, predicted functional effects, and external databases. This makes it possible to move from raw variant lists to meaningful biological interpretation. As I would like to develop skills that are useful in both laboratory and data analysis settings, this topic fits well with my future plans.

<br>

## Planning
### Week 1: Learning the basics
- Learning the theory behind variant annotation.
- Exploring what VCF files are and how annotation works.
- Following a tutorial on one (or more) annotation tool(s) or workflow(s).
- Identifying which software/package to use for my own analysis.

### Week 2: Applying the skill
- Finding a new dataset or example VCF file.
- Annotating the variants using the method/tool.
- Filtering or prioritizing the variants based on functional impact.
- Checking whether the output makes biological sense.

### Week 3: Writing up the assignment
- Cleaning up the code and making it reproducible.
- Writing a short explanation of the workflow and choices.
- Summarizing what I learned and what was challenging.
- Preparing the final report in RMarkdown or another clear format.
